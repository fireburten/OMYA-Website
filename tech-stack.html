philosophy.md

How I Think

Data First, Always

A beautiful dashboard built on bad data is worse than no dashboard. I spend the time understanding the data before touching a visualisation tool.

Simple Until It Needs to Be Complex

A well-written SQL query beats a sprawling ML pipeline when SQL is what the problem needs. I reach for complexity only when simpler tools genuinely can't do the job.

Ask the Uncomfortable Question

The most valuable thing I do is often questioning the question. Most data requests are proxies for something deeper.

Ship, Then Improve

A working dashboard in front of users beats a perfect one that never ships. Get it out, watch how people use it, then make it better. Repeat.
